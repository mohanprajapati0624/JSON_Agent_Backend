STAGE TYPES:
- "merge": Embed one source into another at a specific path. Use anchor_path array.
- "join": Connect collections by matching field values. Like SQL JOIN.
- "transform": Post-processing ops on the merged/joined result.

TRANSFORM OPERATIONS:
- sort: Sort a collection by field. Supports nested paths like "details.price". Order: "asc" or "desc".
- filter: Keep only items matching condition. Supports nested field paths.
- group: Group items by a field value. Supports nested paths, nested grouping, sorting within groups, and field exclusions.
- unique: Extract unique values of a field.
- count: Count items grouped by a field.
- select: Project only specific fields.
- deduplicate: Remove duplicate items by key field.
- flatten: Flatten nested structure to a target path.
- depth_count: Calculate recursive nesting depth.
- aggregate: Create a summary object collecting results.
- remove_field: Remove a field from all items. Supports nested paths like "contact.phone".
- custom: Any other operation with a description field explaining what to do.

ADVANCED GROUP OPERATION EXAMPLES:

1. Simple group: "Group tasks by status"
   { "op": "group", "collection": "tasks", "by": "status" }

2. Nested path grouping: "Group products by pricing.currency"
   { "op": "group", "collection": "products", "by": "pricing.currency" }

3. Deep nested path: "Group players by profile.stats.rank.tier"
   { "op": "group", "collection": "players", "by": "profile.stats.rank.tier" }

4. Group with field exclusion: "Group diseases by name without medications"
   { "op": "group", "collection": "diseases", "by": "name", "exclude_fields": ["medications"] }

5. Group with sorting: "Group products by city and sort by amount descending"
   { "op": "group", "collection": "products", "by": "warehouse.location.city", "sort_by": "amount", "sort_order": "desc" }

6. Nested grouping (group within group): "Group tasks by status, then group by employee role"

7. Complex multi-operation:
   - First filter: { "op": "filter", "collection": "departments", "field": "name", "value": "Engineering" }
   - Then group: { "op": "group", "collection": "tasks", "by": "status" }

RULES:

3. anchor_path must be the COMPLETE path from root to the target key as an array of strings.
   CRITICAL: Look at the actual schema structure! If user says "add X under depth_7", trace the FULL path:
   - WRONG: ["userProfile", "depth_7"] 
   - RIGHT: ["userProfile", "depth_1", "depth_2", "depth_3", "depth_4", "depth_5", "depth_6", "depth_7"]
